# Brightspace Student Hub

Automates data retrieval from D2L Brightspace and optional integrations (Crowdmark, Piazza). Works for any Brightspace institution ‚Äî UWaterloo, UofT, McMaster, etc. All data is fetched via Python scripts using cookies exported from Chrome (CDP). No manual browser interaction needed after initial setup.

## Core Workflow

**Before any task ‚Äî always run setup check first:**
```bash
python3 scripts/check_setup.py
```
If `all_pass` is false, follow `references/onboarding.md` to guide the user through missing steps. Never proceed to data fetching until setup passes.

On first run, `check_setup.py` also auto-discovers the Brightspace instance URL and saves it to `scripts/config.json`. All subsequent scripts read from this config.

Once setup is confirmed:

1. **Load config** ‚Äî read `scripts/config.json` for `base_url`, `timezone`, `integrations`
2. **Check cookies** ‚Äî look for existing cookie files in `scripts/`
3. **Login if needed** ‚Äî use browser-use; Chrome auto-fills saved passwords
4. **Extract cookies** ‚Äî run `export_cookies.py` (CDP `Network.getAllCookies`)
5. **Fetch enrollments** ‚Äî call enrollment API to get current-term course list (never hardcode course IDs)
6. **Execute task** ‚Äî Python scripts call platform APIs using discovered course IDs
7. **Output** ‚Äî display results, convert timestamps to user's local timezone

If any API returns 401 during a task, refresh cookies automatically (see `references/onboarding.md` ‚Üí Cookie Refresh).

## config.json Schema

Generated by `check_setup.py` on first run. Edit manually to add integrations.

```json
{
  "base_url": "https://learn.uwaterloo.ca",
  "institution": "University of Waterloo",
  "timezone": "America/Toronto",
  "term_filter": "Spring 2026",
  "chrome_profile": "/Users/you/Library/Application Support/Codex/LearnChromeProfile",
  "download_dir": "/Users/you/Desktop/brightspace-downloads",
  "integrations": {
    "crowdmark": true,
    "piazza": true
  }
}
```

`term_filter` is matched against enrollment `Name` field ‚Äî set to `null` to get all active courses.

## Course Discovery (Never Hardcode IDs)

Always fetch the course list dynamically at the start of each task:

```python
import json, requests

def get_active_courses(session, config):
    base = config["base_url"]
    resp = session.get(f"{base}/d2l/api/lp/1.26/enrollments/myenrollments/?orgUnitTypeId=3&isActive=1")
    items = resp.json().get("Items", [])
    term = config.get("term_filter")
    if term:
        items = [i for i in items if term in i["OrgUnit"]["Name"]]
    return [{"id": i["OrgUnit"]["Id"], "name": i["OrgUnit"]["Name"]} for i in items]
```

Never hardcode `orgUnitId` values ‚Äî they differ per user and per institution.

## Key Rules

- **Never hardcode course IDs** ‚Äî always discover via enrollment API
- **Never fabricate data** ‚Äî only report what the API returns
- **Always use config timezone** ‚Äî read from `config.json`, default UTC if absent
- **Never overwrite existing files** ‚Äî skip by default; ask before overwriting
- **Confirm before bulk download** ‚Äî before any file download begins, pause and confirm (1) destination path, (2) folder layout (course-first vs type-first), (3) conflict policy (skip/overwrite/rename). Present all three in a single prompt. Do not start downloading until the user explicitly confirms.
- **Generate file trees from disk** ‚Äî `find downloads -type f | sort`, never from logs
- **Unknown institution** ‚Äî if `base_url` domain is not `learn.uwaterloo.ca`, skip Crowdmark/Piazza unless explicitly configured
- **Report issues** ‚Äî if the user encounters a bug, unexpected behaviour, or missing feature, ask: "Would you like me to file a GitHub issue for this?" If they agree, run: `gh issue create --repo WayneWei228/brightspace-student-hub --title "..." --body "..."` with a clear title and description of what went wrong, what was expected, and any relevant error output. Never file an issue without explicit user approval.
